# Applied Computing (VCE study design)

Scope: students apply a problem-solving methodology across data analysis, programming and cyber security, then specialise in Units 3-4 into a Data Analytics and/or a Software Development sequence (students may take one or both).

## Units 1-2 (Year 11)
- **Area of Study 1 — Data analysis (Unit 1):** interpret teacher-provided solution requirements and designs, analyse data and develop data visualisations to present findings. Key knowledge: qualitative/quantitative data types, data quality and Privacy Act/APP considerations, ethical issues (including AI-generated data), relational database structures, spreadsheet functions and descriptive statistics, types/purposes of data visualisations. Key skills: interpret requirements and designs, source data ethically, use APA referencing, apply statistical analysis, build validated databases/spreadsheets/visualisations.
- **Area of Study 2 — Programming (Unit 1):** interpret teacher-provided solution requirements to design and develop a software solution using an object-oriented programming (OOP) language. Key knowledge: emerging AI code-generation trends, data types/structures, OOP principles (abstraction, encapsulation), programming features and control structures, naming conventions, validation/debugging/testing techniques. Key skills: design and build a working solution, apply data structures, document code internally, validate/debug/test.
- **Area of Study 1 — Innovative solutions (Unit 2):** working collaboratively, identify a problem/need/opportunity and analyse, design, develop and evaluate an innovative solution (proof of concept, prototype or product) exploring an emerging technology topic. Key knowledge: functions/impacts of digital systems and emerging technologies, solution specifications, design and project-management tools, relevant legislation (AI Ethics Principles, Copyright, Privacy Acts), ethical issues in AI use. Key skills: investigate a problem, collect data, design and build a solution, test and evaluate it, discuss legal/ethical issues.
- **Area of Study 2 — Cyber security (Unit 2):** respond to a case study to examine a cyber security incident or network vulnerability, evaluate threats, and propose strategies to protect data/information on a network. Key knowledge: emerging cyber security trends, network hardware/software components, security threats/vulnerabilities, frameworks such as the Essential Eight and AI Ethics Principles. Key skills: analyse network security case studies, evaluate threats, propose mitigation strategies with legal/ethical justification.

## Units 3-4 (Year 12)
- **Data Analytics sequence — Area of Study 1 (Unit 3) / Area of Study 1 (Unit 4):** extract, manipulate, cleanse and statistically analyse data from large repositories to build data visualisations against given requirements (Unit 3), then develop the Unit 3-designed solution into infographics/dynamic data visualisations, applying data security and evaluating against criteria (Unit 4). Key knowledge: SQL querying, descriptive statistics and correlation, data visualisation design/formats, data security controls, evaluation strategies. Key skills: extract/validate/cleanse data, apply statistical analysis, build and test visualisations, implement security, evaluate effectiveness.
- **Data Analytics sequence — Area of Study 2 (Unit 3) / Area of Study 2 (Unit 4):** propose a research question, plan and design infographics/visualisations (Unit 3, first part of the School-assessed Task); then respond to a case study on a data breach, evaluating security strategies and recommending improvements (Unit 4). Key skills: formulate a research question, project-plan with Gantt charts, generate/evaluate design ideas; analyse breach impact and recommend security strategies.
- **Software Development sequence — Area of Study 1 (Unit 3) / Area of Study 1 (Unit 4):** use an OOP language to build working software modules against given requirements (Unit 3), then develop the Unit 3 design into a complete, alpha/beta-tested software solution and evaluate it (Unit 4). Key knowledge: OOP principles (abstraction, encapsulation, generalisation, inheritance), data structures, sorting/searching algorithms, error types, debugging/testing techniques, beta-testing strategies. Key skills: build and validate modules, document code, debug/test, conduct beta testing and refine.
- **Software Development sequence — Area of Study 2 (Unit 3) / Area of Study 2 (Unit 4):** independently identify a problem/need/opportunity, plan and design a software solution (Unit 3, first part of the School-assessed Task); then respond to a case study on insecure software development practices, recommending security improvements (Unit 4). Key skills: document a software requirements specification, project-plan and generate design ideas; analyse development-security risks and recommend improvements using frameworks like the Essential Eight.

Cross-study skills: the same problem-solving methodology (analysis, design, development, evaluation) and computational/design/systems thinking recur across all sequences; cyber security and ethical/legal issues (privacy, AI ethics) are woven through every unit.
