# Algorithmics (VCE study design)

Scope: a Units 3-4 only Higher Education Scored Study (HESS) where students model real-world problems with abstract data types, design and analyse algorithms, and explore the foundations of computer science and AI.

## Units 1-2 (Year 11)
- Algorithmics has no Units 1 and 2. This is a Units 3-4 only VCE HESS study, benchmarked to first-year university level; students enrol directly into Unit 3.

## Units 3-4 (Year 12)
- **Area of Study 1 — Data modelling with abstract data types (Unit 3):** define and explain the representation of information using abstract data types (ADTs), and devise formal representations for modelling real-world information problems. Key knowledge: motivation for ADTs, signature specifications, ADTs (set, list, array, dictionary, stack, queue, priority queue, graphs), graph features and categories, decision trees and state graphs. Key skills: explain the role of ADTs, read/write ADT signatures, identify graph properties, select and justify an ADT to model a real-world problem.
- **Area of Study 2 — Algorithm design (Unit 3):** define and explain algorithmic design principles, design algorithms using basic design patterns, and implement them as programs. Key knowledge: algorithm structure, pseudocode, recursion and iteration, brute-force and greedy design patterns, graph traversal (BFS/DFS), graph algorithms (Prim's, Dijkstra's, Bellman-Ford, Floyd-Warshall, PageRank), proof by induction/contradiction. Key skills: interpret and write pseudocode, design recursive/iterative algorithms, select and justify graph algorithms, implement algorithms in a high-level language, demonstrate correctness.
- **Area of Study 3 — Applied algorithms (Unit 3):** design suitable solutions for real-world problems requiring integration of algorithms and data types, and communicate/justify the solution. Key knowledge: characteristics and suitability of ADTs and design patterns, combining ADTs for complex requirements, applying algorithms to real-world problems. Key skills: select and justify fit-for-purpose combinations of ADTs and algorithms, communicate data model/algorithm designs, interpret computed solutions in real-world terms.
- **Area of Study 1 — Formal algorithm analysis (Unit 4):** establish the efficiency of simple algorithms and explain soft limits of computability. Key knowledge: time and space complexity, Big-O notation, recurrence relations and the Master Theorem, P/NP/NP-Hard/NP-Complete classes, combinatorial explosion. Key skills: analyse time efficiency formally, solve recurrence relations, evaluate algorithm suitability by complexity, describe consequences of a problem's complexity class.
- **Area of Study 2 — Advanced algorithm design (Unit 4):** solve information problems using more advanced algorithm design patterns and explain how heuristics address intractability. Key knowledge: binary search, divide and conquer (mergesort, quicksort), dynamic programming, backtracking, heuristics (hill climbing, A*, simulated annealing), graph colouring/knapsack/travelling salesman problems. Key skills: apply and compare design patterns, apply heuristic methods to computationally hard problems, explain heuristic/randomised approaches to intractable problems.
- **Area of Study 3 — Computer science: past and present (Unit 4):** explain the historical context for the emergence of computer science and discuss modern machine learning techniques and the philosophical issues they raise. Key knowledge: Entscheidungsproblem, Turing machines, decidability and the Halting Problem, Turing Test/weak AI/strong AI, Chinese Room Argument, SVMs, neural network structure and training, overfitting/underfitting, AI ethics (transparency, accountability, bias). Key skills: explain the historical origins of computer science, demonstrate hard limits of computability, describe and evaluate AI conceptions and arguments, explain how data-driven algorithms learn, discuss AI ethical issues.

Cross-study skills: strong mathematical and logical reasoning underpins the whole study; Unit 3 Area of Study 3 forms the first part of a School-assessed Task that is completed and extended in Unit 4.
